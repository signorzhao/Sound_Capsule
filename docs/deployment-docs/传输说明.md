# 部署资料目录说明

本目录（`docs/deployment-docs`）是 **Sound Capsule 私有化 Supabase** 的部署资料夹，可与项目一起克隆使用，也可单独拷贝到服务器（如 `/home/sbadmin/`）后按文档操作。

---

## 包含内容

| 类型 | 文件/目录 | 说明 |
|------|-----------|------|
| **文档** | [README.md](./README.md) | 总览与推荐阅读顺序 |
| | [Sound_Capsule_私有化Supabase_完整部署指南.md](./Sound_Capsule_私有化Supabase_完整部署指南.md) | 首次部署一站式步骤 |
| | [私有化Supabase_快速复用部署指南.md](./私有化Supabase_快速复用部署指南.md) | 迁移、备份、恢复 |
| | [部署说明.md](./部署说明.md) | 部署检查清单 |
| | [database/执行顺序.md](./database/执行顺序.md) | SQL 执行顺序（**含 007 上传策略必做**） |
| **脚本** | `deploy-supabase.sh` | 首次部署 |
| | `backup-supabase.sh` / `restore-supabase.sh` | 备份与恢复 |
| **SQL** | `database/` | 建表与 Storage 策略（含 **007**） |
| **配置参考** | `docker-compose.yml` 等 | Supabase 编排；可选参考 CHANGELOG.md、versions.md |

---

## 使用方式

1. **在项目内使用**：直接阅读 [README.md](./README.md)，按推荐顺序操作。
2. **拷贝到服务器**：将本目录（或压缩包）传到服务器，例如 `/home/sbadmin/deployment-docs`，部署脚本与 SQL 路径以该目录为准（如 `database/supabase_schema.sql`）。
3. **SQL 顺序**：以 [database/执行顺序.md](./database/执行顺序.md) 为准，**务必执行 007**（Storage 上传策略），否则上传会 403。

更多细节（Auth、故障排查）见项目 **[docs/SUPABASE_PRIVATE_DEPLOY.md](../SUPABASE_PRIVATE_DEPLOY.md)**。  
应用更新记录见 **[docs/应用更新记录.md](../应用更新记录.md)**。
