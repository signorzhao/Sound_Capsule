# 如何查看测试结果（简单版）

## 🎯 快速方法

### 在网页上查看（最简单）

1. **打开这个链接**：
   ```
   https://github.com/signorzhao/Sound_Capsule/actions
   ```

2. **点击左侧的 "Build and Test Windows"**

3. **点击最新的运行**（最上面的）

4. **查看这些步骤的输出**：
   - ✅ **Test Application - Check Files** → 看文件是否找到
   - ✅ **Test Application - Run with Timeout** → 看应用是否运行
   - ✅ **Test Application - Check Logs** → 看错误信息

5. **下载测试结果**（页面最底部）：
   - 点击 `App-Log-{数字}` 下载日志文件
   - 点击 `Test-Results-{数字}` 下载测试文件

---

## 📊 如何解读测试结果

### ✅ 成功的标志

在 "Test Application - Check Files" 步骤中看到：
```
✓ SoundCapsule.exe 找到: ...
✓ capsules_api.exe 找到: ...
```

在 "Test Application - Run with Timeout" 步骤中看到：
```
应用已启动，PID: ...
运行中... (2/30 秒)
运行中... (4/30 秒)
...
应用仍在运行，正常结束进程...
```

在 "Test Application - Check Logs" 步骤中看到：
```
✓ 日志文件存在
✓ 未发现明显错误
```

### ❌ 失败的标志

**文件未找到**：
```
✗ SoundCapsule.exe 未找到
✗ capsules_api.exe 未找到
```
→ **问题**：构建失败或文件路径错误

**应用闪退**：
```
应用已退出（运行了 X 秒）
```
→ **问题**：应用启动后立即崩溃

**日志文件不存在**：
```
⚠️ 日志文件不存在
```
→ **问题**：应用未启动或启动失败

**发现错误**：
```
⚠️ 发现错误信息:
  [错误内容]
```
→ **问题**：查看具体错误内容

## 🔍 如何找到错误原因

### 步骤 1：查看日志步骤

在 "Test Application - Check Logs" 步骤中，会显示：
- 日志文件的最后 50 行
- 所有包含 "error"、"失败"、"异常" 的行

### 步骤 2：下载日志文件

1. 在运行页面底部找到 "Artifacts"
2. 下载 `App-Log-{数字}` 
3. 解压后查看 `export_debug.log` 文件

### 步骤 3：查看 Windows 事件日志

在 "Test Application - Check Windows Event Log" 步骤中，会显示系统级的错误信息。

## 💡 常见问题

### Q: 截图里显示的是什么？

A: 那是 GitHub Actions Runner 的系统日志，不是测试结果。需要看的是测试步骤的输出。

### Q: 在哪里看测试结果？

A: 在 GitHub 网页上：
1. 进入 Actions 页面
2. 选择 "Build and Test Windows"
3. 点击最新的运行
4. 查看各个测试步骤的输出

### Q: 应用闪退了怎么办？

A: 
1. 查看 "Test Application - Check Logs" 步骤
2. 下载 `App-Log` Artifact
3. 查看日志文件中的错误信息
4. 检查 "Test Application - Check Windows Event Log" 步骤

### Q: 文件找不到怎么办？

A: 
1. 查看 "Test Application - Extract" 步骤，看解压后的文件结构
2. 查看 "Package" 步骤，看文件是否正确复制
3. 检查构建步骤是否有错误
